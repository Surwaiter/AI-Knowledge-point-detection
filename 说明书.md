# ProjectLearn 技术说明书

## 一、项目概述

ProjectLearn 是一个基于 C++/Win32 API 开发的 Windows 桌面知识学习与评测系统。它能够导入多种格式的文档（TXT、PDF、DOCX、PPTX、XLSX 等），利用大语言模型（LLM）自动分析文档内容，构建三级知识体系分类，并通过 RAG（检索增强生成）技术实现智能问答与知识掌握度评估。

### 核心技术栈

| 层级 | 技术选型 |
|------|----------|
| 语言 | C++17 |
| GUI | 原生 Win32 API（无第三方 UI 框架） |
| HTTP | WinHTTP（Windows 原生 HTTP 客户端） |
| 数据库 | SQLite3（嵌入式数据库） |
| 文档解析 | 自研解析器（ZIP/DEFLATE/PDF 解析） |
| AI 接口 | OpenAI 兼容 API（支持多家 LLM 提供商） |
| 向量检索 | 内存向量存储 + 余弦相似度搜索 |
| 文本处理 | 自研 JSON 解析器、Markdown↔RTF 转换 |

### 系统要求

- 操作系统：Windows 7 及以上（64 位）
- 运行环境：Visual C++ 2019+ 运行时库
- 外部依赖：SQLite3（已内置 sqlite3.c）

## 三、核心数据结构

### 3.1 知识体系（三级分类）

```
KnowledgeCategory (类别)
├── category: string          — 类别名称
├── lastUpdated: string       — 最后更新日期
└── chapters: [KnowledgeChapter]
    ├── title: string         — 章节标题
    ├── summary: string       — 章节摘要
    └── items: [KnowledgeItem]
        ├── name: string      — 知识点名称
        ├── content: string   — 知识点内容
        └── sourceFile: string — 来源文件
```

### 3.2 掌握度评估

```
MasteryRecord
├── knowledgePoint: string    — 知识点名称
├── level: int               — 掌握等级 (0-4)
├── averageScore: double     — 最近平均得分
├── totalAttempts: int       — 总尝试次数
├── recentCorrect: int       — 最近正确次数
├── recentTotal: int         — 最近总次数
├── lastUpdated: string      — 最后更新时间
└── recentScores: int[]      — 最近得分记录 (最多 10 条)
```

**掌握等级定义：**

| 等级 | 名称 | 颜色 | 条件 |
|------|------|------|------|
| 0 | 未掌握 | #E74C3C (红) | averageScore < 30 |
| 1 | 了解 | #E67E22 (橙) | 30 ≤ averageScore < 60 |
| 2 | 理解 | #F1C40F (黄) | 60 ≤ averageScore < 75 |
| 3 | 熟练 | #2ECC71 (绿) | 75 ≤ averageScore < 90 |
| 4 | 精通 | #1ABC9C (青) | averageScore ≥ 90 |

### 3.3 问答记录

```
QARecord
├── knowledgePoint: string    — 关联知识点
├── question: string         — 问题内容
├── userAnswer: string       — 用户回答
├── score: int              — 得分 (0-100)
├── level: string           — 等级文字
└── timestamp: string       — 记录时间
```

## 五、核心功能流程

### 5.1 文件分析流程

```
1. 用户选择文件 → FileHandler 读取文件内容
2. OfficeExtractor / PdfTextExtractor 提取文本
   ├── DOCX: ZIP 解压 → word/document.xml → 结构化段落提取（含标题层级、表格、图片）
   ├── PDF: 流对象解析 → DEFLATE/ASCII85 解码 → BT/ET 文本提取
   ├── TXT: 直接读取
   └── 其他: 原始字节扫描
3. LLMClient::AnalyzeFileContent()
   → 发送文本到 LLM，要求提取：
     - 学科领域分类
     - 章节划分
     - 知识点及内容
     - 知识点考察题目（含答案和评分标准）
4. LLM 返回 JSON → 解析为 KnowledgeCategory[] 
5. Storage::MergeAndSaveCatalog() → 合并到知识目录
6. RAGEngine::IndexDocument() → 文本分块 → 向量化 → 存入向量存储
7. KnowledgeDB::SaveKnowledgePoints() → SQLite 持久化
```

### 5.2 智能问答流程（RAG）

```
1. 用户输入问题 + 选择知识点范围
2. RAGEngine::BuildRAGQuery()
   ├── EmbeddingClient::GetEmbedding(question) → 获取问题向量
   ├── VectorStore::Search(vector, topK) → 余弦相似度检索 top-K 相关文本块
   └── 构建提示词 = 系统指令 + 检索到的上下文 + 用户问题
3. LLMClient::Chat() → 发送提示词到 LLM
4. LLM 返回答案文本 + 掌握度评估 JSON
5. KnowledgeEngine::ParseVerificationResult() → 解析得分和等级
6. Storage::UpdateMastery() → 更新掌握度记录
7. 显示结果给用户
```

### 5.3 知识评测流程

```
1. 用户选择类别/章节/知识点 + 测试模式（顺序/随机/易错点）
2. KnowledgeEngine::FilterByMode() → 根据模式筛选题目
3. 解析范围表达式 ParseRangeExpr("1-40;43-50") → 生成题目索引
4. 显示题目 → 用户作答 → 提交
5. KnowledgeEngine::VerifyItemAnswer() → LLM 评分（0-100）
6. Storage::UpdateMastery() → 更新掌握度
7. Storage::SaveQARecord() → 保存问答记录
```

## 六、LLM API 集成

### 6.1 支持的接口

| 接口 | 用途 | 端点格式 |
|------|------|----------|
| Chat Completions | 对话/分析/评测 | `{api_endpoint}/chat/completions` |
| Embeddings | 文本向量化 | `{api_endpoint}/embeddings` |

### 6.2 内置 LLM 提供商预设

| 提供商 | API 端点 | 默认模型 |
|--------|----------|----------|
| DeepSeek | `https://api.deepseek.com` | `deepseek-chat` |
| 通义千问 (Qwen) | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-plus` |
| 智谱 (GLM) | `https://open.bigmodel.cn/api/paas/v4` | `glm-4-flash` |
| 月之暗面 (Moonshot) | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` |
| 百度千帆 (ERNIE) | `https://qianfan.baidubce.com/v2` | `ernie-speed-128k` |
| OpenAI | `https://api.openai.com` | `gpt-3.5-turbo` |
| 自定义 | 用户自行填写 | 用户自行填写 |

### 6.3 关键 Prompt 模板

**文件分析 Prompt**（从文档提取知识体系）：
- 角色：知识体系分析专家
- 输出：严格的 JSON 格式
- 结构：`[{category, chapters: [{title, summary, knowledge_points: [{name, content, test_questions: [{question, answer, score_criteria}]}]}]}]`
- 要求：题目数量需合理（长文档 50+ 题）、中文输出、内容详细

**问答评估 Prompt**（评估用户回答）：
- 角色：知识掌握度评估专家
- 输出：`{score: 0-100, level: "未掌握/了解/理解/熟练/精通", comment: "评语"}`

**评估报告生成 Prompt**：
- 根据用户所有 QA 历史记录和掌握度数据，生成综合性学习评估报告

## 七、文本处理技术

### 7.1 RAG 文本分块（TextChunker）

```
参数:
- chunkSize: 2048 字符
- overlapSize: 256 字符
- 策略: 优先按段落边界切分，段落过大时按句子边界切分
- 单 chunk 最大行数: 20 行（对代码等紧凑文本限制）
```

### 7.2 向量嵌入（EmbeddingClient）

```
- 模型: text-embedding-3-small (OpenAI 兼容)
- 向量维度: 由 API 返回决定（通常 1536）
- 解析: 手动解析 JSON float 数组
- 请求格式: {model, input, encoding_format: "float"}
```

### 7.3 向量存储（SimpleVectorStore）

```
- 存储结构: 内存 vector<pair<vector<float>, string>>
- 检索算法: 余弦相似度（内积/模长之积）
- 持久化: JSON 文件序列化
- Top-K: 默认 5，最大 10
```

### 7.4 文档解析器

**OfficeExtractor（OOXML 格式）**：
- ZIP 文件结构解析（Local File Header 遍历）
- DEFLATE 压缩解压（自研 Huffman 树解码器）
- XML 文本提取（w:t / a:t / t / v 标签）
- DOCX 结构化提取：
  - 标题层级检测（大纲级别 / 样式 / 字体大小 / 文本模式）
  - 表格提取（Markdown 表格格式）
  - 图片提取与关联（rels 关系解析）
  - 目录检测与提取
  - 代码块检测（等宽字体识别）
  - 列表项检测（numPr 编号属性）
- OLE (.doc) 格式解析（FAT/MiniFAT/目录结构）

**PdfTextExtractor（PDF 格式）**：
- 二进制直接解析（无第三方库依赖）
- 流对象定位与提取
- 多过滤器解码：FlateDecode / ASCII85Decode / ASCIIHexDecode
- 文本提取：BT/ET 块中的 Tj/TJ 操作符
- 元数据提取（Title/Author/Subject 等）
- Unicode（UTF-16BE）和 PDF 转义字符解码
- UTF-8 字节清理与规范化

## 十二、已知限制

1. **向量存储**：仅支持内存存储，大规模文档可能占用大量内存
2. **PDF 解析**：不支持需要 JBIG2/JPX/DCT 解码的 PDF 文件
3. **并发**：LLM 调用为同步阻塞（使用 WinHTTP），不支持并发请求
4. **API 响应**：依赖 LLM 返回严格 JSON 格式，格式偏差可能导致解析失败
5. **文件大小**：解压输出限制 64MB（Office）/ 8MB（PDF），超大文件可能截断
6. **国际化**：仅支持中文和英文混合内容，未做多语言适配
